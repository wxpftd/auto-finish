Daytona is not running here

This is the auto-finish daytona-stub — an nginx placeholder. The real Daytona OSS is a separate multi-service stack and is not bundled into this docker-compose. The orchestrator will fail to create sandboxes until you bring up a real Daytona server.

Install local Daytona OSS

git clone https://github.com/daytonaio/daytona
cd daytona/docker
docker compose up -d

By default, Daytona's API listens on port 3986. Once it's up, stop this stub:

docker compose -f /path/to/auto-finish/docker-compose.yml stop daytona-stub

Then point the orchestrator at the real server by editing .env:

DAYTONA_API_URL=http://localhost:3986
DAYTONA_API_KEY=<token from daytona dashboard>

Or use Daytona Cloud

Set DAYTONA_API_URL to the cloud endpoint and DAYTONA_API_KEY to your cloud API token, then stop the stub.

See Daytona OSS docs.
